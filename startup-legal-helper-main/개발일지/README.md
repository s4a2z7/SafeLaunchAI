# 📚 개발일지 명세서

모듈러박스 프로젝트의 성장 과정을 날짜별로 기록합니다.

---

## 📖 개발일지 작성 규칙

### 파일명 규칙
| 항목 | 내용 |
|------|------|
| 형식 | `YYYY-MM-DD.md` |
| 예시 | `2026-01-22.md` |
| 인코딩 | UTF-8 |

### 작성 원칙
1. **길게 쓰지 않는다** - 나중에 검색했을 때 이해할 수 있을 정도면 충분
2. **삽질 기록을 빠뜨리지 않는다** - 같은 실수 반복 방지를 위해
3. **"왜(Why)"를 기록한다** - 무엇을 했는지보다 왜 그렇게 했는지
4. **매일 작성한다** - 작은 것이라도 기록하는 습관

## 📋 일지 템플릿

새로운 개발일지를 작성할 때는 아래 템플릿을 사용하세요:

```markdown
# 📅 개발 일지: [오늘의 주요 작업 키워드]
- 날짜: YYYY-MM-DD
- 태그: #관련태그 #기술스택 #도메인

---

## 🎯 오늘의 목표
- [ ] 목표 1
- [ ] 목표 2

## 🛠 작업 내용
- **진행 사항:** 
  - 무엇을 했는지 구체적으로

- **사용한 도구:** 

## 🚧 삽질 기록 (Troubleshooting)
> 개발 중 마주친 에러나 어려움을 기록합니다.

### 문제점
- 어떤 문제가 발생했는지

### 해결 방법
- 어떻게 해결했는지
- AI에게 어떤 질문을 했는지

## 💡 새로 알게 된 점 (TIL)
- 오늘 배운 것들

## 📌 넥스트 스텝
- [ ] 내일 할 일 1
- [ ] 내일 할 일 2

---

## 💭 회고
오늘의 작업에 대한 간단한 생각 정리
```

## 🔍 검색 팁

### VS Code에서 검색하기
1. `Ctrl + Shift + F` (전체 검색)
2. 검색 위치를 `개발일지/` 폴더로 지정
3. 키워드나 태그로 검색

### 자주 찾는 내용
| 찾고 싶은 것 | 검색어 |
|-------------|--------|
| 에러 해결 | `#에러` 또는 `🚧 삽질 기록` |
| 학습 내용 | `💡 새로 알게 된 점` |
| 특정 기술 | 태그 검색 (예: `#React`, `#Python`) |

## 💡 개발일지 활용법

### 1. 주간 회고
- 매주 일요일, 한 주간의 일지를 읽으며 무엇을 배웠는지 정리
- 반복되는 문제가 있다면 해결책 마련

### 2. 월간 리뷰
- 매월 말, 한 달간의 성장을 확인
- 목표 대비 달성도 점검
- 다음 달 계획 수립

### 3. 포트폴리오 자료
- 개발일지는 그 자체로 성장의 증거
- 면접이나 프로젝트 설명 시 활용 가능
